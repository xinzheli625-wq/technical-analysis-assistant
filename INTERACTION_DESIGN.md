# 人机交互设计：全流程交互点与实现

## 核心理念

你不是"使用"一个工具，而是**与一个技术分析助手协作**。

- **系统负责**：精确计算、自动匹配、初步分析、统计追踪
- **你负责**：审核结果、指导学习、关键决策、纠正偏差

---

## 18个交互点详解

### 交互点 ①：配置（初始化）

---

### 交互点 ②：请求分析

**你做什么**：输入股票代码和分析参数

```python
from api import analyze
result = analyze("AAPL", days=100)  # 自动保存记录
```

**系统做什么**：
1. 下载数据（Yahoo Finance）
2. FeatureExtractor 计算50+指标
3. MarketRegimeDetector 判断市场环境
4. SkillMatcher 匹配触发的Skill（包括准触发）
5. 构建Prompt（指标数据 + 触发Skill清单 + 准触发Skill清单 + 未触发Skill清单）
6. LLM 基于Prompt做Phase 1/2/3/4分析

### 交互点 ④：追问系统（质疑与澄清）

**场景**：你对某个结论有疑问

**Claude Code 对话方式**：
> "为什么RSI超买警戒没有被触发？"

系统回应: "RSI超买警戒的触发条件是 RSI > 70。
当前RSI = 67.3，距离阈值还差2.7点。
根据准触发定义（差20%阈值），67.3/70=96%，属于准触发范围。
系统已将其标记为'准触发'并在Phase 2中提示注意。"

**Claude Code 对话方式**：
> "为什么RSI接近超买不算一个更强的看空信号？"

系统回应: "根据[Skill: LLM裁决指南]规则1：
'当动量信号与超买信号冲突时，若超买仅为准触发，以动量信号优先。'
原因：准触发意味着条件未完全满足，信号强度较弱。
当前MACD金叉是明确触发（柱状线扩张），量价突破是明确触发（量比2.3>2）。
两个强看涨信号 vs 一个弱看空准触发 → 看涨优先。"

### 交互点 ⑦：上传资料（教系统新方法）

**你做什么**：提供学习材料

**Claude Code 对话方式**：
> "上传这本书的 Skill"（拖拽PDF到对话中）

或

> "我学到一种方法：当成交量放大3倍以上且价格突破前高时，应该买入..."

**Python 直接调用**：
```python
from api import assistant

# 方式1：上传书籍
result = assistant().upload_skill_book("专业投机原理.pdf")

# 方式2：从文本上传
result = assistant().upload_skill_text("""
[新方法名称] 成交量验证突破法

核心思想：价格突破时，必须伴随成交量放大，否则假突破概率高。

分析步骤：
1. 识别关键阻力位或支撑位
2. 观察价格是否突破该位
3. 检查突破时的成交量与20日均量之比
4. 若量比>2，确认有效突破；若量比<1.5，警惕假突破
5. 结合RSI判断是否存在超买/超卖背离

参考数据：
- 量比>2：明显放量
- 量比1.5-2：温和放量
- 量比<1.5：缩量

常见误区：
- 只看价格突破，不看成交量
- 在震荡市中应用此法则（震荡市中假突破多）

不适用场景：
- 震荡市（布林带宽度<5%）
- 一字板/无量涨停

适用市场环境：
- trending_up
- trending_down
""")
```

**系统做什么**：
1. 解析文本（PDF/Word/直接输入）
2. LLM 提取结构化Skill教材：
   ```json
   {
     "name": "成交量验证突破法",
     "type": "methodology",
     "core_idea": "价格突破必须伴随成交量放大...",
     "analysis_steps": ["步骤1...", "步骤2...", ...],
     "reference_data": {"量比>2": "明显放量", ...},
     "common_pitfalls": ["只看价格突破，不看成交量", ...],
     "when_not_to_use": ["震荡市", "一字板"],
     "applicable_regimes": ["trending_up", "trending_down"],
     "win_rate_hint": {"trending_up": 0.75, "ranging": 0.45}
   }
   ```
3. 存储为 **pending** 状态（等你审核）

## 关键设计决策

### 1. Skill条件语言：复杂版

支持：
- 指标数值比较：`rsi > 70`, `volume_ratio > 2`
- 指标间比较：`rsi > macd_hist * 100`
- 时间序列：`连续3天 RSI > 70`
- 复合逻辑：`A AND B OR (C AND D)`

### 2. 准触发阈值：差20%

```
触发条件: RSI > 70
当前值: 67.3
差距: (70-67.3)/70 = 3.86% ← 小于20%，算准触发

触发条件: MACD金叉
当前值: DIF=0.23, DEA=0.15 → 已金叉
差距: 0% ← 已触发
```

### 3. 协同规则：从Skill文档学习

不硬编码协同规则，而是：
- 从Skill教材中自然语言描述提取协同关系
- 从反馈数据中统计哪些Skill组合同时触发时胜率高
- LLM在Phase 3中基于Skill教材和统计数据自行推断协同

### 4. LLM裁决指南：独立Skill

```json
{
  "name": "LLM裁决指南",
  "type": "system",
  "core_idea": "当多个Skill信号冲突时，如何裁决",
  "analysis_steps": [
    "步骤1：识别所有冲突的信号对",
    "步骤2：比较各信号的历史胜率（分当前市场环境）",
    "步骤3：评估各信号的触发强度（明确触发 > 准触发）",
    "步骤4：评估市场环境偏好（趋势市优先趋势信号，震荡市优先震荡信号）",
    "步骤5：给出裁决结果和理由"
  ],
  "reference_data": {
    "裁决优先级": "明确触发 > 准触发 > 未触发",
    "环境偏好": "趋势市: 趋势信号 > 反转信号；震荡市: 反转信号 > 趋势信号"
  }
}
```

这是一个特殊的系统级Skill，每次分析时自动注入Prompt中，指导LLM如何裁决冲突。

## 交互方式总结

| 交互点 | 你的角色 | 系统角色 | 交互方式 |
|--------|---------|---------|---------|
| ① 配置 | 设置参数 | 保存 | CLI命令 |
| ② 请求分析 | 输入代码 | 全自动计算分析 | CLI命令 |
| ③ 查看报告 | 阅读理解 | 输出结构化报告 | CLI输出 |
| ④ 追问 | 提出疑问 | 展开解释 | CLI命令/chat |
| ⑤ 确认/纠错 | 标记对错 | 记录反馈 | CLI命令 |
| ⑥ 交易决策 | 自主决策 | 无 | 人自主 |
